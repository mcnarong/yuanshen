Full Feature Map

系统总览

以下内容不是宣传语，而是基于当前源码、配置与资源文件实际存在的能力清单。

Combat Core

七元素与反应流水线

支持元素附着、附着量、持续时间、标准 ICD、元素精通加成、怪物抗性、减抗上限与最终伤害结算。

  • 七元素基础倍率与反应倍率都可写表达式。
  • 支持自定义变量、PlaceholderAPI 和 exp4j 公式解析。
  • 冻结衰减、感电跳电、燃烧持续、草原核视觉都可配置。
Party System

角色编队与当前出战槽位

角色系统不再把武器塞进 GUI，而是专注 4 槽角色编队，战斗时直接检测玩家主手。

  • 支持 1 到 4 号角色槽位和当前选中槽。
  • 切换角色后可自动尝试换到匹配武器。
  • 若当前槽无有效角色，会自动归位到第一个有效槽。
Weapon Layer

武器、精炼与被动效果

武器系统支持定义、发放、识别、等级、精炼和战斗被动，并记忆每个角色上次使用的武器。

  • 只检测主手是否为已注册原神武器。
  • 当前角色与武器类型匹配时进入角色战斗模式。
  • 支持法器、单手剑、双手剑、长柄武器与弓类型。
Skill Engine

统一角色技能引擎

E、Q、普攻、重击、部分角色被动和运行时状态都接入统一战斗链路。

  • /ys skill e 与 /ys skill q 可直接触发技能。
  • 近战角色走插件普攻/重击逻辑，弓角色保留各自实现。
  • 技能释放失败会检查当前角色、武器匹配、冷却和能量。
Constellation

命星与命座升级系统

已接入命星物品、命座升级、总览 GUI、命座文本与部分命座效果的真实战斗联动。

  • 命星可发放、可消耗、可统计当前数量。
  • 角色命座上限 6 命。
  • 角色配置里可覆盖命座名称与描述。
Visual Layer

侧边栏、伤害显示与占位符

展示层不是附属功能，而是已形成可持续配置的面板体系。

  • 玩家级侧边栏开关、伤害显示开关。
  • 默认、战斗、防御三套侧边栏模板。
  • 58 个主占位符覆盖属性、角色、武器、冷却、元素与抗性。
Operations

指令、管理员 GUI 与配置重载

服主可以通过命令行和 GUI 双入口管理玩家、武器、角色、显示状态与当前元素。

  • 支持在线玩家选择、武器发放、数据查看、刷新属性。
  • 管理员 GUI 内可切侧边栏、切伤害显示、清元素。
  • 支持运行时配置重载。
Integration

MythicMobs 与 Mod 按键桥

插件能与外部内容系统协作，而不是封闭运行。

  • MythicMobs 可按 mythic-mob-id 匹配怪物抗性。
  • 客户端按键桥支持 1-4 槽切人、E、Q、打开 GUI。
  • 绑定有激活模式、冷却和是否吞原版按键的控制。
Persistence

玩家数据持久化

离线重连和插件重载之后，关键角色系统状态可以恢复。

  • 角色槽、当前选中槽、武器记忆、技能能量与冷却都会保存。
  • 侧边栏开关、伤害显示开关、当前元素会保存。
  • 部分角色运行时状态也会持久化恢复。
Battle Framework

战斗框架与反应细节

插件的核心不是单个技能，而是完整的伤害链路：属性来源、元素附着、反应计算、怪物抗性、展示反馈与数据同步。

16 种已支持反应

反应不仅包含基础倍率，还包含消耗量、冷却窗口、范围、二次附着、护盾与草原核实体配置。

增幅反应

蒸发、融化

转化反应

超载、感电、超导、碎冰、扩散

状态反应

冻结、燃烧、结晶

激化与草系链路

原激化、超激化、蔓激化、绽放、超绽放、烈绽放

  • 默认标准 ICD 为 2.5 秒 / 3 hit，可配置起始偏移。
  • 附着分普攻、重击、E、Q、下落五类强度入口。
  • 元素精通对不同反应有单独权重，而不是统一倍率。

战斗结算轨迹

01
读取属性

角色基础属性 + 武器属性/被动 + Lore 属性 + 配置默认值。

02
判定当前模式

只有已选角色存在，且主手武器与角色类型匹配时，角色战斗模式才激活。

03
附着与 ICD

按动作组、命中窗与附着量决定是否挂元素及其持续时间。

04
触发反应

处理消耗量、额外伤害、冻结与燃烧状态、草原核、护盾、扩散与减抗。

05
应用怪物抗性

优先 MythicMobs ID，匹配失败时回退原版实体类型抗性。

06
输出展示

战斗提示、伤害显示、侧边栏刷新、粒子/能量状态同步与持久化保存。

能量与产球

角色能量、Q 最大能量、最近一次产球来源、粒子数量、持续时间都可读出并展示到侧边栏或 PlaceholderAPI。

  • 支持 %yuanshen_particle_status% 一类的最近产球状态占位符。
  • 侧边栏会自动补入“最近产球”这一实用行。
  • Yelan、Hu Tao、Xiangling 等角色已经接入对应技能能量流程。

属性覆盖到原版实体

可选把原神计算后的生命、攻击、防御映射回 Bukkit 原版属性，用于与原版机制共存。

  • 可同步 MAX_HEALTH、ATTACK_DAMAGE。
  • 可选把防御折算到 ARMOR，默认关闭。
  • 支持仅在角色模式激活时才接管原版属性。

Lore 属性解析

可从主手和护甲 Lore 读取攻击、防御、生命、暴击、元素伤害、元素抗性和物理伤害等属性。

  • 默认读取主手和四件护甲。
  • 默认关闭副手读取。
  • 支持“全元素伤害”“全元素抗性”等聚合属性。

Control Surface

命令系统、权限节点与 GUI

命令不是演示级接口，而是已经支撑服主管理、玩家操作和运维维护的正式入口。

插件控制

/ys reload
  • 重载主配置、角色配置、属性配置与侧边栏配置。
  • 重载武器定义、角色定义和怪物抗性。
  • 在线玩家会刷新属性、侧边栏与按键桥状态。

属性与展示

/ys stats [玩家]
  • 输出当前角色槽、当前角色、当前武器、当前元素。
  • 输出攻击、防御、生命、暴击率、暴伤、精通、充能等数值。
  • 胡桃等角色的特定运行状态也会追加展示。

伤害显示

/ys damage <toggle|on|off> [玩家]
  • 切换自己或他人的伤害显示。
  • 设置结果会立即保存到 playerdata.yml。
  • 支持自用权限与他人管理权限分离。

侧边栏显示

/ys sidebar <toggle|on|off> [玩家]
  • 切换自己或他人的侧边栏显示。
  • 会触发实时刷新或卸载侧边栏记分板。
  • 设置同样会持久化保存。

角色技能

/ys skill <e|q>
  • 通过命令直接释放 E 或 Q。
  • 自动检测当前角色、武器匹配、冷却、能量与目标。
  • 成功施放后会重置普攻连段等角色运行状态。

角色编队 GUI

/ys open
  • 打开角色编队界面。
  • 只允许放角色物品，不存武器。
  • 点击槽位下方按钮可切换出战角色并查看命座。

角色与武器列表

/ys list <character|weapon>
  • 读取当前已注册角色或武器定义列表。
  • 用于发放前确认 ID、别名与资源是否加载成功。

发放角色 / 武器 / 命星

/ys give character <角色> [等级] [命座] [玩家] /ys give weapon <武器> [等级] [精炼] [玩家] /ys give constellation [数量] [玩家]
  • 支持按定义上限自动钳制等级、命座和精炼。
  • 发放失败会检查背包空间与目标玩家是否在线。

出战角色切换

/ys player set <1|2|3|4>
  • 切换当前角色槽位。
  • 保存当前槽位到玩家数据。
  • 切换后刷新属性、侧边栏和武器匹配状态。

管理员 GUI

/ys admin
  • 打开腐竹管理主菜单。
  • 可进入玩家列表、玩家详情、武器发放与配置重载入口。

当前元素快捷设置

/ys <火|水|冰|雷|风|岩|草|清除>
  • /ys 火 这种写法会自动按元素子命令处理。
  • 设置结果会刷新侧边栏并保存到玩家数据。

权限节点

插件在 plugin.yml 中声明了 14 个权限节点，默认值已经按玩家自用与 OP 管理拆开。

yuanshen.reload重载配置，默认 op
yuanshen.stats.self查看自己的属性，默认 true
yuanshen.stats.others查看他人属性，默认 op
yuanshen.damage.self切自己的伤害显示，默认 true
yuanshen.damage.others切他人伤害显示，默认 op
yuanshen.sidebar.self切自己的侧边栏，默认 true
yuanshen.sidebar.others切他人侧边栏，默认 op
yuanshen.character.gui打开角色编队，默认 true
yuanshen.character.give发放角色物品，默认 op
yuanshen.player.set切换出战槽位，默认 true
yuanshen.skill.cast命令施法，默认 true
yuanshen.element.self设置当前元素，默认 true
yuanshen.weapon.give发放武器，默认 op
yuanshen.admin.gui打开管理员 GUI，默认 op

角色编队 GUI

27 格界面，核心布局包含 4 个角色槽、4 个状态按钮、帮助卡、状态卡、模式卡和切换说明。

  • 提示当前槽位、当前角色、主手武器状态和所需武器类型。
  • 槽位按钮会根据是否选中、是否就绪、是否为空切换材质。
  • 支持通过配置修改标题、布局、材质、自定义模型值和文案。

命座 GUI

展示角色当前命座、下一命效果、命星消耗与总览信息，是角色成长的可视化入口。

  • 显示角色当前等级、命座与全部已解锁命座说明。
  • 若命星不足会阻止升级。
  • 若槽位没有角色，会给出明确阻断提示。

管理员 GUI

服主不需要只靠命令行。管理员 GUI 已经覆盖玩家查看、武器发放和部分运维操作。

  • 主菜单支持玩家管理、配置重载、关闭面板。
  • 玩家页支持角色/武器编辑、武器发放、数据查看、刷新属性。
  • 还可切侧边栏、切伤害显示、清除当前元素。
Display · Bridge · Config

展示系统、联动接口与配置面

这个插件的“可用性”来自完整的展示层和配置层，而不只是技能代码本身。

侧边栏系统

默认提供 default、combat、defense 三套模板，支持标题、刷新频率、默认开关和占位符行配置。

  • 全局开关：sidebar.enabled。
  • 玩家默认是否开启：sidebar.default-enabled。
  • 会自动补入武器信息和最近产球状态。

PlaceholderAPI 扩展

安装 PlaceholderAPI 后，插件会注册自己的扩展。属性、武器、角色、冷却、粒子、抗性都能读。

  • 支持角色名、角色槽、元素键、武器类型、武器精炼。
  • 支持 E/Q 冷却文本、秒数和毫秒值。
  • 支持七元素加成和七元素抗性，外加物伤与物抗。

Mod 按键桥

通过插件消息通道把客户端按键映射到服务端命令，减少玩家必须敲命令的割裂感。

  • 默认绑定：1/2/3/4 切人，E 战技，Q 爆发，L 打开 GUI。
  • 支持 always、active_character、yuanshen_weapon、compatible_weapon 四种激活模式。
  • 支持每个绑定独立冷却和 consume-vanilla 行为。

MythicMobs 桥接

若服务端安装 MythicMobs，插件会反射读取活动怪物实例的 mobType 来匹配抗性配置。

  • 桥接初始化失败时会退回原版实体类型匹配。
  • 怪物抗性支持全元素、单元素、物理与减抗限制。

软依赖与兼容面

当前声明的软依赖有 PlaceholderAPI、ProtocolLib 与 MythicMobs。

  • 当前源码中的实际桥接重点在 PlaceholderAPI 与 MythicMobs。
  • 项目说明明确当前版本不再依赖 MMOItems。

公式配置能力

主配置不仅是简单数字，而是表达式系统。

  • 元素倍率、反应倍率、冻结时长、聚怪范围等都可写数学表达式。
  • 支持 custom_variables 二次复用。
  • 可结合 PlaceholderAPI 注入玩家等级等变量。

Runtime Safety

玩家数据持久化与工程信息

这个插件已经开始像一个长期维护项目一样处理状态保存、配置迁移、异步落盘和构建产物。

保存文件与生命周期

  • 玩家数据保存到 plugins/Yuanshen/playerdata.yml。
  • 玩家加入时加载，退出时保存并卸载内存状态。
  • 插件关闭时会保存所有在线玩家后再清理缓存。
  • 采用延迟写入 + 异步落盘，减少频繁 IO。

持久化内容

  • 4 个角色槽位、当前选中槽位、角色槽内物品。
  • 角色对应的武器记忆。
  • 每个角色的能量、E 冷却时间、Q 冷却时间。
  • 伤害显示开关、侧边栏开关、当前元素。
  • 胡桃 E 状态、额外伤害、支援 Buff、重击冷却。
  • 香菱辣椒 Buff 等部分运行时状态。

当前明确不持久化的部分

  • 实体身上的临时元素附着。
  • 血梅香目标跟踪等部分纯运行时对象。
  • 某些正在持续演算中的临时实体状态。

构建与运行要求

  • Java 21
  • Paper 1.21+，构建 API 为 1.21.4-R0.1-SNAPSHOT
  • Gradle ShadowJar 输出 build/libs/Yuanshen-1.0.jar
  • 可用构建命令：./gradlew.bat compileJava、./gradlew.bat clean shadowJar

工程成熟度判断

  • 框架层成熟：角色、武器、编队、命座、GUI、持久化、展示层都已成体系。
  • 玩法层持续打磨：元素反应还原度与单个角色细节仍在 Beta 迭代。
  • 当前最适合定位为测试服 / 体验服 / 小规模玩法服核心战斗框架。
