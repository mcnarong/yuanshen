Feature Map

功能总览系统能力、战斗链路、反应触发规则与联动接口

这一页负责讲清楚“插件到底实现了什么”，尤其是反应系统如何判定。 这里写的是基于当前源码与配置整理出的实际行为，不是泛泛而谈的宣传词。

阅读建议

  • 想知道系统边界：看“系统构成”
  • 想知道反应怎么打：看“反应触发规则”
  • 想知道怎么实际操作：转到 Wiki 玩家教程
Framework Layers

系统构成

插件当前最成熟的是整套框架层，而不仅仅是元素反应本身。

Combat Core

元素与伤害系统

  • 七元素基础倍率可配置。
  • 反应倍率、消耗量、持续时间均可表达式化。
  • 支持元素精通、元素伤害加成与抗性结算。
Party

角色编队系统

  • 支持 4 槽角色位和当前选中槽。
  • 角色 GUI 只存角色，不存武器。
  • 切人后刷新属性、武器状态和面板。
Weapon

武器与精炼系统

  • 支持武器定义、发放、等级、精炼。
  • 只检测主手是否为注册武器。
  • 支持角色切换后记忆武器。
Skill Engine

统一技能引擎

  • 角色 E/Q 通过统一入口施放。
  • 近战与弓系逻辑分开实现。
  • 可由命令或按键桥触发。
Display

面板与展示层

  • 侧边栏支持多布局模板。
  • PlaceholderAPI 可读角色、武器、冷却、粒子、抗性。
  • 玩家级伤害显示和侧边栏开关会持久化。
Bridge

联动与桥接

  • MythicMobs 可按 mobType 桥接怪物抗性。
  • ModKeybindBridge 可把客户端按键映射为服务端命令。
  • 配置支持表达式、变量和部分 PlaceholderAPI 注入。
Battle Flow

战斗链路

插件的反应不是单点 if-else，而是完整流水线。

结算顺序

01
读取属性

角色基础属性 + 武器属性/被动 + Lore 属性 + 默认配置。

02
判定角色模式

当前角色存在且主手武器匹配时，进入角色战斗模式。

03
附着与 ICD

按动作组和标准 ICD 规则决定是否挂元素。

04
反应规划

源码会先计算冻结、激化、双元素共存等状态，再决定下一步反应。

05
反应伤害与状态

处理增幅伤害、范围伤害、冻结、燃烧、草原核、护盾与减抗。

06
展示与保存

刷新侧边栏、发送反应文本、更新玩家运行时数据。

附着与 ICD 默认规则

  • 标准 ICD 默认开启，时间窗 2500ms，计数窗 3 hit，起始偏移 0。
  • 附着强度分为普攻、重击、E、Q、下落攻击五个入口。
  • 普通附着持续时间默认 200 tick。
  • 冻结、感电、燃烧、扩散和草原核还有各自的附加参数。
Reaction Rules

反应触发规则

下面写的是“怎么触发”，不是只列反应名称。

蒸发

当目标已有 火附着，再用 水 攻击，会触发蒸发，并且在当前源码里走更强的一档倍率与更高消耗量。反过来，如果目标已有 水附着，再用 火 攻击，也会触发蒸发，但走较弱的一档。

融化

当目标已有 冰附着，再用 火 攻击，会触发较强一档融化。若目标已有 火附着，再用 冰 攻击，也会触发融化，但倍率较低。

超载

火与雷相遇时触发。无论是 火打雷 还是 雷打火，都可进入超载流程，并在目标周围造成爆炸范围伤害。

感电

水 + 雷 互相作用触发。触发后目标会进入持续放电状态，并按配置间隔结算跳电；还可向附近目标传导次级感电。

冻结

水 + 冰 互相作用触发。触发后目标进入冻结状态，并同时携带水/冰两种载体。冻结不是纯静态状态，它有独立的衰减模型和碎冰阈值。

碎冰

在本插件当前源码里，物理伤害 或 岩元素 打到冻结目标时，满足冻结量阈值即可触发碎冰，造成额外物理伤害。

超导

冰 + 雷 互相作用触发。除额外冰伤外，还会对范围内目标施加物理减抗，持续时间和减抗值都在配置里可调。

原激化 / 超激化 / 蔓激化

雷 + 草 先触发原激化，使目标进入激化状态。之后如果再吃到 雷，会触发超激化；如果再吃到 草，会触发蔓激化。源码里激化状态会单独跟踪并优先处理。

绽放 / 超绽放 / 烈绽放

水 + 草 先生成草原核。之后若草原核被 雷 触发，则进入超绽放，生成追踪型伤害；若被 火 触发，则进入烈绽放，造成范围爆炸伤害。

燃烧

火 + 草 互相作用触发。目标进入燃烧状态后，会按 tick 周期持续掉血，并继续消耗草附着、刷新火附着，还会向近处扩散燃烧火附着。

扩散

风 不能直接扩散所有元素。当前源码只会从目标身上的 火 / 水 / 雷 / 冰 四种可扩散元素中选一个主附着进行扩散，并向范围内目标施加扩散伤害与二次附着。

结晶

岩 与目标身上的 火 / 水 / 雷 / 冰 任一附着反应时触发。会掉落对应元素的结晶碎片，拾取后生成护盾；同元素护盾吸收可按配置获得额外倍率。

Advanced Cases

源码里的优先级与特殊情况

这部分专门解释为什么有时反应看起来不是“简单二选一”。

感电双载体优先级

  • 若目标同时携带 水 + 雷，源码会先把它视为感电载体状态。
  • 这时若再吃 火，优先从雷这边继续触发超载。
  • 若再吃 冰，优先从雷这边触发超导。
  • 若再吃 草，优先从雷这边推进到原激化。
  • 若再吃 风 / 岩，也会优先围绕雷载体处理扩散 / 结晶。

冻结目标被火命中时

  • 冻结目标同时带有水和冰两种载体。
  • 当前源码会比较两者附着量，决定先走 融化 还是先走 蒸发。
  • 如果冰附着足够突出，会优先先吃冰做融化。
  • 如果水附着更占主导，也可能先吃水做蒸发。
Display & Bridge

展示层与联动能力

功能不止在战斗本体，展示与配置层同样已经成型。

侧边栏系统

  • 支持 default / combat / defense 三套布局。
  • 会自动插入武器信息和最近产球状态。
  • 可玩家级开关并持久化保存。

PlaceholderAPI

  • 覆盖角色、武器、冷却、粒子、伤害、抗性。
  • 适合菜单插件、计分板和展示面板对接。

Mod 按键桥

  • 默认绑定 1-4 槽切人、E、Q、L。
  • 支持激活模式、冷却和吞原版按键。
