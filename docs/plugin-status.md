# Yuanshen 插件系统现状评估

更新日期: 2026-04-06

## 一句话结论

这个项目现在已经不只是元素反应插件，而是一个带有角色、武器、命座、技能、元素战斗、GUI、持久化和管理面板的原神战斗框架。

如果以服主视角判断，它已经适合做测试服、体验服、小规模私服或 Beta 展示服。
如果要作为公开正式服的核心卖点，目前还不建议宣传成“完整还原原神”。

## 适合正常服主使用的部分

- 角色系统
  - 角色定义、角色物品、等级、命座、属性成长、别名解析都已接入。
- 编队与切人系统
  - 支持 4 槽角色位、当前选中槽、角色切换、角色模式判定。
- 武器系统
  - 支持武器定义、等级、精炼、武器被动、角色切换后记忆武器。
- 命座系统
  - 支持命星道具、命座升级、命座描述、命座 GUI。
- 数据持久化
  - 玩家角色槽、武器记忆、显示开关、部分技能运行时状态会存档与恢复。
- 指令与管理功能
  - 已有 `/ys reload/stats/damage/sidebar/skill/open/give/list/player/admin`。
- GUI 系统
  - 已有角色编队 GUI、命座 GUI、管理员 GUI。
- 展示系统
  - 已有侧边栏、PlaceholderAPI 扩展、伤害显示开关。

这些部分已经不是“演示性质”，而是可以直接配置、发放、管理、维护的系统。

## 可以宣传，但应写成 Beta 的部分

- 元素伤害系统
  - 已有元素伤害、暴击、元素精通、元素伤害加成、抗性结算。
- 元素反应系统
  - 已有附着、附着量、持续时间、标准 ICD、冻结、感电、燃烧、激化、绽放、超绽放、烈绽放、扩散、结晶、护盾、多阶段反应流水线。
- 技能系统
  - 已有角色技能引擎，普攻、重击、下落、E、Q 都已经接到统一战斗链路。
- 能量系统
  - 已有元素微粒、充能效率、Q 能量获取、充能提示。
- 怪物抗性系统
  - 已支持按怪物配置元素抗性与物理抗性，也支持与 MythicMobs 对接。
- Mod 按键桥
  - 已支持把客户端按键映射到槽位切换、E、Q 等操作。

这些功能已经能用，但更适合宣传为：

- 原神战斗框架 Beta
- 原神角色战斗体验服
- 支持元素反应与部分角色技能的测试版

## 暂不建议当正式版本宣传的部分

- 不建议宣传为“完整还原原神战斗系统”
  - 反应系统虽然已经很强，但仍处于持续重构阶段，复杂联动还需要继续实机验证和细调。
- 不建议宣传为“内容量丰富”
  - 目前内容数量仍偏少，角色、武器、怪物配置都还不多。
- 不建议宣传为“适合公开大服长期稳定运行”
  - 当前更接近研发中的 Beta，而不是经过大量线上压力验证的正式版。
- 不建议宣传为“所有角色都高度还原”
  - 目前是少量角色已进入可玩状态，不是完整角色库。

## 面向服主的实际建议

### 适合的定位

- 测试服
- 体验服
- 小规模私服
- 开发中的 Beta 展示服

### 不适合的定位

- 公开正式服的核心宣传卖点
- 高强度长期运营的大服核心战斗插件
- 宣传为“完整原神复刻”

## 推荐宣传口径

如果现在要对外描述，建议使用下面这类表述：

> 一个正在持续开发中的原神战斗框架 Beta。  
> 已支持角色编队、武器、命座、元素反应、部分角色技能与战斗面板展示。  
> 适合测试服、体验服和小规模玩法服使用。

不建议使用下面这类表述：

> 完整还原原神全部战斗机制  
> 高度还原所有角色  
> 可直接用于大型正式服长期稳定运行

## 最终判断

从工程状态看，这个插件最成熟的是“角色/武器/编队/命座/GUI/持久化/管理工具”这一整套框架。

从玩法状态看，最值得继续打磨的是“元素反应还原度”和“单个角色技能细节”。

所以现在最准确的定位是：

**框架已经成熟，玩法内容仍在快速迭代。**
